> 日期：2026-06-15 | 版本：v0.2 | 变更：商业模式重构

## 变更原因

1. **降低获客摩擦** — 免费 = 零门槛试用，不付费就能体验全部功能
2. **规避收费合规** — 教育付费涉及发票/退款/税务，一人公司初期不碰
3. **用户更自由** — 选哪个 AI、花多少钱，用户自己决定
4. **和 OneCode 一致** — OneCode 本身就是"自带 API Key"模式，一脉相承

## 架构变更

### 之前

```
用户 → 我们的服务器 → 我们的 AI API（付费用户才有 Key）
```

### 现在

```
用户 → 我们的前端 → 用户的 AI API（用户自己的 Key）
                  ↘ 我们的题库（免费 CDN）
```

**关键变化**：AI 调用从服务端移到客户端，我们不经手 API Key。

### 技术方案

| 方案 | 说明 | 选择 |
|------|------|------|
| A. 纯前端调用 | JS 直接调 AI API | ⚠️ 浏览器 CORS 限制 |
| B. 轻量代理 | 我们的 server 只做请求转发，不存 Key | ✅ 推荐 |
| C. 本地应用 | Electron/小程序本地调 | ✅ 小程序方案 |

**推荐方案 B**：用户在前端输入 Key → 请求带 Key 到我们的轻量代理 → 代理转发到 AI API → 原路返回。代理不存 Key，不记日志。

## 商业模式重构

### 不靠产品收费，靠什么活？

| 阶段 | 收入来源 | 说明 |
|------|----------|------|
| 现在 | 无（先验证价值） | 免费=最快验证 Problem-Solution Fit |
| 有用户后 | 生态收入 | API 厂商返佣 / 推荐注册返利 |
| 有规模后 | 增值服务 | 高级功能（企业版/培训机构版/出题 API） |
| 长期 | 开源+托管 | 开源代码免费，托管版收服务费 |

### 核心逻辑

```
免费 → 用户多 → 数据多 → 产品更好 → 用户更多 → 增值服务变现

不靠"锁功能收费"，靠"用的人多后找到变现点"
```

## 对 OKR 的影响

| 原KR | 调整后 | 原因 |
|------|--------|------|
| KR3: 付费用户 10 个 | → 日活 100+（不追付费，追使用） | 没有付费墙，用活跃度代替 |
